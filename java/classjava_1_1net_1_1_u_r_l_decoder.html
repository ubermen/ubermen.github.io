Detailed Description

Utility class for HTML form decoding. This class contains static methods for decoding a String from the application/x-www-form-urlencoded MIME format.

The conversion process is the reverse of that used by the URLEncoder class. It is assumed that all characters in the encoded string are one of the following: "

  • A sequence of the form "<i>@code %xy @endcode </i>" will be treated as representing a byte where xy is the two-digit hexadecimal representation of the 8 bits. Then, all substrings that contain one or more of these byte sequences consecutively will be replaced by the character(s) whose encoding would result in those consecutive bytes. The encoding scheme used to decode these characters may be specified, or if unspecified, the default encoding of the platform will be used.

There are two possible ways in which this decoder could deal with illegal strings. It could either leave illegal characters alone or it could throw an java.lang.IllegalArgumentException. Which approach the decoder takes is left to the implementation.

Decodes a

application/x-www-form-urlencoded

string using a specific encoding scheme. The supplied encoding is used to determine what characters are represented by any consecutive sequences of the form "<i>@code %xy @endcode </i>".

Note: The World Wide Web Consortium Recommendation states that UTF-8 should be used. Not doing so may introduce incompatibilities.
