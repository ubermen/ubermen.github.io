25 class GoogleCloudStorageToGoogleCloudStorageOperator(BaseOperator):
26  """
27  Copies objects from a bucket to another, with renaming if requested.
28 
29  :param source_bucket: The source Google cloud storage bucket where the
30  object is. (templated)
31  :type source_bucket: string
32  :param source_object: The source name of the object to copy in the Google cloud
33  storage bucket. (templated)
34  If wildcards are used in this argument:
35  You can use only one wildcard for objects (filenames) within your
36  bucket. The wildcard can appear inside the object name or at the
37  end of the object name. Appending a wildcard to the bucket name is
38  unsupported.
39  :type source_object: string
40  :param destination_bucket: The destination Google cloud storage bucket
41  where the object should be. (templated)
42  :type destination_bucket: string
43  :param destination_object: The destination name of the object in the
44  destination Google cloud storage bucket. (templated)
45  If a wildcard is supplied in the source_object argument, this is the
46  prefix that will be prepended to the final destination objects' paths.
47  Note that the source path's part before the wildcard will be removed;
48  if it needs to be retained it should be appended to destination_object.
49  For example, with prefix ``foo/*`` and destination_object `'blah/``, the
50  file ``foo/baz`` will be copied to ``blah/baz``; to retain the prefix write
51  the destination_object as e.g. ``blah/foo``, in which case the copied file
52  will be named ``blah/foo/baz``.
53  :type destination_object: string
54  :param move_object: When move object is True, the object is moved instead
55  of copied to the new location.
56  This is the equivalent of a mv command as opposed to a
57  cp command.
58  :type move_object: bool
59  :param google_cloud_storage_conn_id: The connection ID to use when
60  connecting to Google cloud storage.
61  :type google_cloud_storage_conn_id: string
62  :param delegate_to: The account to impersonate, if any.
63  For this to work, the service account making the request must have
64  domain-wide delegation enabled.
65  :type delegate_to: string
66 
67  **Examples**:
68  The following Operator would copy a single file named
69  ``sales/sales-2017/january.avro`` in the ``data`` bucket to the file named
70  ``copied_sales/2017/january-backup.avro` in the ``data_backup`` bucket ::
71  copy_single_file = GoogleCloudStorageToGoogleCloudStorageOperator(
72  task_id='copy_single_file',
73  source_bucket='data',
74  source_object='sales/sales-2017/january.avro',
75  destination_bucket='data_backup',
76  destination_object='copied_sales/2017/january-backup.avro',
77  google_cloud_storage_conn_id=google_cloud_conn_id
78  )
79 
80  The following Operator would copy all the Avro files from ``sales/sales-2017``
81  folder (i.e. with names starting with that prefix) in ``data`` bucket to the
82  ``copied_sales/2017`` folder in the ``data_backup`` bucket. ::
83  copy_files = GoogleCloudStorageToGoogleCloudStorageOperator(
84  task_id='copy_files',
85  source_bucket='data',
86  source_object='sales/sales-2017/*.avro',
87  destination_bucket='data_backup',
88  destination_object='copied_sales/2017/',
89  google_cloud_storage_conn_id=google_cloud_conn_id
90  )
91 
92  The following Operator would move all the Avro files from ``sales/sales-2017``
93  folder (i.e. with names starting with that prefix) in ``data`` bucket to the
94  same folder in the ``data_backup`` bucket, deleting the original files in the
95  process. ::
96  move_files = GoogleCloudStorageToGoogleCloudStorageOperator(
97  task_id='move_files',
98  source_bucket='data',
99  source_object='sales/sales-2017/*.avro',
100  destination_bucket='data_backup',
101  move_object=True,
102  google_cloud_storage_conn_id=google_cloud_conn_id
103  )
104  """
