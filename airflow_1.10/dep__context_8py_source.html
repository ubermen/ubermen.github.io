32 class DepContext(object):
33  """
34  A base class for contexts that specifies which dependencies should be evaluated in
35  the context for a task instance to satisfy the requirements of the context. Also
36  stores state related to the context that can be used by dependendency classes.
37 
38  For example there could be a SomeRunContext that subclasses this class which has
39  dependencies for:
40  - Making sure there are slots available on the infrastructure to run the task instance
41  - A task-instance's task-specific dependencies are met (e.g. the previous task
42  instance completed successfully)
43  - ...
44 
45  :param deps: The context-specific dependencies that need to be evaluated for a
46  task instance to run in this execution context.
47  :type deps: set(BaseTIDep)
48  :param flag_upstream_failed: This is a hack to generate the upstream_failed state
49  creation while checking to see whether the task instance is runnable. It was the
50  shortest path to add the feature. This is bad since this class should be pure (no
51  side effects).
52  :type flag_upstream_failed: boolean
53  :param ignore_all_deps: Whether or not the context should ignore all ignoreable
54  dependencies. Overrides the other ignore_* parameters
55  :type ignore_all_deps: boolean
56  :param ignore_depends_on_past: Ignore depends_on_past parameter of DAGs (e.g. for
57  Backfills)
58  :type ignore_depends_on_past: boolean
59  :param ignore_in_retry_period: Ignore the retry period for task instances
60  :type ignore_in_retry_period: boolean
61  :param ignore_task_deps: Ignore task-specific dependencies such as depends_on_past and
62  trigger rule
63  :type ignore_task_deps: boolean
64  :param ignore_ti_state: Ignore the task instance's previous failure/success
65  :type ignore_ti_state: boolean
66  """
