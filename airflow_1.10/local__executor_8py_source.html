19 """
20 LocalExecutor runs tasks by spawning processes in a controlled fashion in different
21 modes. Given that BaseExecutor has the option to receive a `parallelism` parameter to
22 limit the number of process spawned, when this parameter is `0` the number of processes
23 that LocalExecutor can spawn is unlimited.
24 
25 The following strategies are implemented:
26 1. Unlimited Parallelism (self.parallelism == 0): In this strategy, LocalExecutor will
27 spawn a process every time `execute_async` is called, that is, every task submitted to the
28 LocalExecutor will be executed in its own process. Once the task is executed and the
29 result stored in the `result_queue`, the process terminates. There is no need for a
30 `task_queue` in this approach, since as soon as a task is received a new process will be
31 allocated to the task. Processes used in this strategy are of class LocalWorker.
32 
33 2. Limited Parallelism (self.parallelism > 0): In this strategy, the LocalExecutor spawns
34 the number of processes equal to the value of `self.parallelism` at `start` time,
35 using a `task_queue` to coordinate the ingestion of tasks and the work distribution among
36 the workers, which will take a task as soon as they are ready. During the lifecycle of
37 the LocalExecutor, the worker processes are running waiting for tasks, once the
38 LocalExecutor receives the call to shutdown the executor a poison token is sent to the
39 workers to terminate them. Processes used in this strategy are of class QueuedLocalWorker.
40 
41 Arguably, `SequentialExecutor` could be thought as a LocalExecutor with limited
42 parallelism of just 1 worker, i.e. `self.parallelism = 1`.
43 This option could lead to the unification of the executor implementations, running
44 locally, into just one `LocalExecutor` with multiple modes.
45 """

75  """
76  Executes command received and stores result state in queue.
77  :param key: the key to identify the TI
78  :type key: Tuple(dag_id, task_id, execution_date)
79  :param command: the command to execute
80  :type command: string
81  """
82  if key is None:
83  return

87  subprocess.check_call(command, shell=True, close_fds=True)

91  self.log.error("Failed to execute task %s.", str(e))

101 class QueuedLocalWorker(LocalWorker):
102 
103  """LocalWorker implementation that is waiting for tasks from a queue and will
104  continue executing commands as they become available in the queue. It will terminate
105  execution once the poison token is found."""
106 
107  def __init__(self, task_queue, result_queue):
108  super(QueuedLocalWorker, self).__init__(result_queue=result_queue)
109  self.task_queue = task_queue

111  def run(self):
112  while True:
113  key, command = self.task_queue.get()
114  if key is None:
115  # Received poison pill, no more tasks to run
116  self.task_queue.task_done()
117  break
118  self.execute_work(key, command)
119  self.task_queue.task_done()
120  time.sleep(1)
