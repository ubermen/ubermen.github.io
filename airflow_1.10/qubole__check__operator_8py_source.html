27 class QuboleCheckOperator(CheckOperator, QuboleOperator):
28  """
29  Performs checks against Qubole Commands. ``QuboleCheckOperator`` expects
30  a command that will be executed on QDS.
31  By default, each value on first row of the result of this Qubole Commmand
32  is evaluated using python ``bool`` casting. If any of the
33  values return ``False``, the check is failed and errors out.
34 
35  Note that Python bool casting evals the following as ``False``:
36 
37  * ``False``
38  * ``0``
39  * Empty string (``""``)
40  * Empty list (``[]``)
41  * Empty dictionary or set (``{}``)
42 
43  Given a query like ``SELECT COUNT(*) FROM foo``, it will fail only if
44  the count ``== 0``. You can craft much more complex query that could,
45  for instance, check that the table has the same number of rows as
46  the source table upstream, or that the count of today's partition is
47  greater than yesterday's partition, or that a set of metrics are less
48  than 3 standard deviation for the 7 day average.
49 
50  This operator can be used as a data quality check in your pipeline, and
51  depending on where you put it in your DAG, you have the choice to
52  stop the critical path, preventing from
53  publishing dubious data, or on the side and receive email alerts
54  without stopping the progress of the DAG.
55 
56  :param qubole_conn_id: Connection id which consists of qds auth_token
57  :type qubole_conn_id: str
58 
59  kwargs:
60 
61  Arguments specific to Qubole command can be referred from QuboleOperator docs.
62 
63  :results_parser_callable: This is an optional parameter to
64  extend the flexibility of parsing the results of Qubole
65  command to the users. This is a python callable which
66  can hold the logic to parse list of rows returned by Qubole command.
67  By default, only the values on first row are used for performing checks.
68  This callable should return a list of records on
69  which the checks have to be performed.
70 
71  .. note:: All fields in common with template fields of
72  QuboleOperator and CheckOperator are template-supported.
73  """

119 class QuboleValueCheckOperator(ValueCheckOperator, QuboleOperator):
120  """
121  Performs a simple value check using Qubole command.
122  By default, each value on the first row of this
123  Qubole command is compared with a pre-defined value.
124  The check fails and errors out if the output of the command
125  is not within the permissible limit of expected value.
126 
127  :param qubole_conn_id: Connection id which consists of qds auth_token
128  :type qubole_conn_id: str
129 
130  :param pass_value: Expected value of the query results.
131  :type pass_value: str/int/float
132 
133  :param tolerance: Defines the permissible pass_value range, for example if
134  tolerance is 2, the Qubole command output can be anything between
135  -2*pass_value and 2*pass_value, without the operator erring out.
136 
137  :type tolerance: int/float
