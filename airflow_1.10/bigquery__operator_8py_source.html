29  """
30  Executes BigQuery SQL queries in a specific BigQuery database
31 
32  :param bql: (Deprecated. Use `sql` parameter instead) the sql code to be
33  executed (templated)
34  :type bql: Can receive a str representing a sql statement,

36  Template reference are recognized by str ending in '.sql'.
37  :param sql: the sql code to be executed (templated)
38  :type sql: Can receive a str representing a sql statement,
39  a list of str (sql statements), or reference to a template file.
40  Template reference are recognized by str ending in '.sql'.
41  :param destination_dataset_table: A dotted
42  (<project>.|<project>:)<dataset>.<table> that, if set, will store the results
43  of the query. (templated)
44  :type destination_dataset_table: string
45  :param write_disposition: Specifies the action that occurs if the destination table
46  already exists. (default: 'WRITE_EMPTY')
47  :type write_disposition: string
48  :param create_disposition: Specifies whether the job is allowed to create new tables.
49  (default: 'CREATE_IF_NEEDED')
50  :type create_disposition: string
51  :param allow_large_results: Whether to allow large results.
52  :type allow_large_results: boolean
53  :param flatten_results: If true and query uses legacy SQL dialect, flattens
54  all nested and repeated fields in the query results. ``allow_large_results``
55  must be ``true`` if this is set to ``false``. For standard SQL queries, this
56  flag is ignored and results are never flattened.
57  :type flatten_results: boolean
58  :param bigquery_conn_id: reference to a specific BigQuery hook.
59  :type bigquery_conn_id: string
60  :param delegate_to: The account to impersonate, if any.
61  For this to work, the service account making the request must have domain-wide
62  delegation enabled.
63  :type delegate_to: string
64  :param udf_config: The User Defined Function configuration for the query.
65  See https://cloud.google.com/bigquery/user-defined-functions for details.

187  """
188  Creates a new, empty table in the specified BigQuery dataset,
189  optionally with schema.
190 
191  The schema to be used for the BigQuery table may be specified in one of
192  two ways. You may either directly pass the schema fields in, or you may
193  point the operator to a Google cloud storage object name. The object in
194  Google cloud storage must be a JSON file with the schema fields in it.
195  You can also create a table without schema.
196 
197  :param project_id: The project to create the table into. (templated)
198  :type project_id: string
199  :param dataset_id: The dataset to create the table into. (templated)
200  :type dataset_id: string
201  :param table_id: The Name of the table to be created. (templated)
202  :type table_id: string
203  :param schema_fields: If set, the schema field list as defined here:
204  https://cloud.google.com/bigquery/docs/reference/rest/v2/jobs#configuration.load.schema
205 
206  **Example**: ::
207 
208  schema_fields=[{"name": "emp_name", "type": "STRING", "mode": "REQUIRED"},
209  {"name": "salary", "type": "INTEGER", "mode": "NULLABLE"}]
210 
211  :type schema_fields: list
212  :param gcs_schema_object: Full path to the JSON file containing
213  schema (templated). For
214  example: ``gs://test-bucket/dir1/dir2/employee_schema.json``
215  :type gcs_schema_object: string
216  :param time_partitioning: configure optional time partitioning fields i.e.
217  partition by field, type and expiration as per API specifications.
218 
219  .. seealso::
220  https://cloud.google.com/bigquery/docs/reference/rest/v2/tables#timePartitioning
221  :type time_partitioning: dict
222  :param bigquery_conn_id: Reference to a specific BigQuery hook.
223  :type bigquery_conn_id: string
224  :param google_cloud_storage_conn_id: Reference to a specific Google
225  cloud storage hook.
226  :type google_cloud_storage_conn_id: string
227  :param delegate_to: The account to impersonate, if any. For this to
228  work, the service account making the request must have domain-wide
229  delegation enabled.
230  :type delegate_to: string
231

233 
234  CreateTable = BigQueryCreateEmptyTableOperator(
235  task_id='BigQueryCreateEmptyTableOperator_task',
236  dataset_id='ODS',
237  table_id='Employees',
238  project_id='internal-gcp-project',
239  gcs_schema_object='gs://schema-bucket/employee_schema.json',
240  bigquery_conn_id='airflow-service-account',
241  google_cloud_storage_conn_id='airflow-service-account'
242  )
243 
244  **Corresponding Schema file** (``employee_schema.json``): ::
245 
246  [
247  {
248  "mode": "NULLABLE",
249  "name": "emp_name",
250  "type": "STRING"
251  },
252  {
253  "mode": "REQUIRED",
254  "name": "salary",
255  "type": "INTEGER"
256  }
257  ]
258 
259  **Example (with schema in the DAG)**: ::
260 
261  CreateTable = BigQueryCreateEmptyTableOperator(
262  task_id='BigQueryCreateEmptyTableOperator_task',
263  dataset_id='ODS',
264  table_id='Employees',
265  project_id='internal-gcp-project',
266  schema_fields=[{"name": "emp_name", "type": "STRING", "mode": "REQUIRED"},
267  {"name": "salary", "type": "INTEGER", "mode": "NULLABLE"}],

330 class BigQueryCreateExternalTableOperator(BaseOperator):
331  """
332  Creates a new external table in the dataset with the data in Google Cloud
333  Storage.
334 
335  The schema to be used for the BigQuery table may be specified in one of
336  two ways. You may either directly pass the schema fields in, or you may
337  point the operator to a Google cloud storage object name. The object in
338  Google cloud storage must be a JSON file with the schema fields in it.
339 
340  :param bucket: The bucket to point the external table to. (templated)
341  :type bucket: string
342  :param source_objects: List of Google cloud storage URIs to point
343  table to. (templated)
344  If source_format is 'DATASTORE_BACKUP', the list must only contain a single URI.
345  :type object: list
346  :param destination_project_dataset_table: The dotted (<project>.)<dataset>.<table>
347  BigQuery table to load data into (templated). If <project> is not included,
348  project will be the project defined in the connection json.
349  :type destination_project_dataset_table: string
350  :param schema_fields: If set, the schema field list as defined here:

352

354

364  :type source_format: string
365  :param compression: [Optional] The compression type of the data source.

384  columns are treated as bad records, and if there are too many bad records, an

387  :type allow_jagged_rows: bool
388  :param bigquery_conn_id: Reference to a specific BigQuery hook.
389  :type bigquery_conn_id: string
390  :param google_cloud_storage_conn_id: Reference to a specific Google
391  cloud storage hook.
392  :type google_cloud_storage_conn_id: string
393  :param delegate_to: The account to impersonate, if any. For this to
394  work, the service account making the request must have domain-wide
