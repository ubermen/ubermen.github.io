93 def policy(task_instance):
94  """
95  This policy setting allows altering task instances right before they
96  are executed. It allows administrator to rewire some task parameters.
97 
98  Note that the ``TaskInstance`` object has an attribute ``task`` pointing
99  to its related task object, that in turns has a reference to the DAG
100  object. So you can use the attributes of all of these to define your
101  policy.
102 
103  To define policy, add a ``airflow_local_settings`` module
104  to your PYTHONPATH that defines this ``policy`` function. It receives
105  a ``TaskInstance`` object and can alter it where needed.
106 
107  Here are a few examples of how this can be useful:
108 
109  * You could enforce a specific queue (say the ``spark`` queue)
110  for tasks using the ``SparkOperator`` to make sure that these
111  task instances get wired to the right workers
112  * You could force all task instances running on an
113  ``execution_date`` older than a week old to run in a ``backfill``
114  pool.
115  * ...
116  """
