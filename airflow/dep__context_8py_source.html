25 class DepContext(object):
26  """
27  A base class for contexts that specifies which dependencies should be evaluated in
28  the context for a task instance to satisfy the requirements of the context. Also
29  stores state related to the context that can be used by dependendency classes.
30 
31  For example there could be a SomeRunContext that subclasses this class which has
32  dependencies for:
33  - Making sure there are slots available on the infrastructure to run the task instance
34  - A task-instance's task-specific dependencies are met (e.g. the previous task
35  instance completed successfully)
36  - ...
37 
38  :param deps: The context-specific dependencies that need to be evaluated for a
39  task instance to run in this execution context.
40  :type deps: set(BaseTIDep)
41  :param flag_upstream_failed: This is a hack to generate the upstream_failed state
42  creation while checking to see whether the task instance is runnable. It was the
43  shortest path to add the feature. This is bad since this class should be pure (no
44  side effects).
45  :type flag_upstream_failed: boolean
46  :param ignore_all_deps: Whether or not the context should ignore all ignoreable
47  dependencies. Overrides the other ignore_* parameters
48  :type ignore_all_deps: boolean
49  :param ignore_depends_on_past: Ignore depends_on_past parameter of DAGs (e.g. for
50  Backfills)
51  :type ignore_depends_on_past: boolean
52  :param ignore_in_retry_period: Ignore the retry period for task instances
53  :type ignore_in_retry_period: boolean
54  :param ignore_task_deps: Ignore task-specific dependencies such as depends_on_past and
55  trigger rule
56  :type ignore_task_deps: boolean
57  :param ignore_ti_state: Ignore the task instance's previous failure/success
58  :type ignore_ti_state: boolean
59  """
